#服务service

#查看服务的状态
service 服务名 status
例如:
service network status
service iptables status
#启动服务
service 服务名 start
例如:
service network start
service iptables start
#停止服务
service 服务名 stop
例如:
service network stop
service iptables stop
#重启服务
service 服务名 restart
例如:
service network restart
#linux中一切皆文件,服务相关的文件都存放在/etc/init.d/服务名
#etc目录下存放的是配置信息
例如: 
vim /etc/init.d/iptables
vim /etc/init.d/network
#通过chkconfig命令设置自启动
#查看服务chkconfig --list|grep 服务名
#chkconfig --level 5 服务名 on/off
#linux的运行级别
#查看默认级别
vim /etc/inittab
#常用的是级别3和5
#不能设置为2,6
#linux运行级别
0:系统停机状态，不能正常启动
1:但用户状态，只能root权限登陆，禁止远程登陆
2:多用户状态，不支持网络
3:多用户状态，登录后进入控制台命令模式
4:系统未使用,保留
5:x11控制台，登录后进入图形gui模式，默认级别
6:系统正常开启，然后自动关闭，重启，一直重复。


#端口netstat
-an 按一定顺序输出
-p 显示进程
查看那个进程占用了指定端口
netstat -anp|grep 8080

#进程ps
ps -a全部的
ps -u用户的格式显示
ps -x显示参数
查看java相关进程，可以查看
%CPU:进程占用CPU的百分比
%MEM:进程占用物理内存的百分比
VSZ:进程占用的虚拟内存大小
RSS:进程占用的物理内存大小
ps -aux|grep java

查看java进程的pid和ppid，这样我们可以根据pid杀死该进程
ps -ef|grep java
-e：所有进程
-f:全格式

杀死进程
kill
kill pid 通过pid杀死进程
kill -9 pid 强制杀死进程
killall name 通过进程的名字杀死进程

#linux的运行级别runlevel
查看linux的运行级别
runlevel
注意:
我们最常使用的是3和5
3是命令行窗口，
5是x11有界面的
不能设置为0和6，这样不能正常启动linux

暂时切换linux的运行级别:重启以后回复默认 
init 级别
例如:init 3/init 5

永久切换linux的运行级别:重启后生效
vim /etc/inittab

网络network
桥接模式:客户机与主机的ip在同一个网段，外网可以访问客户机，与主机是同一个级别。
NAT模式:客户机与主机单独开一个子网，这时主机在两个网段都有ip，客户机和主机可以单独通信，但是外网不能访问客户机。
：修改linux的网络1、通过修改文件2、通过界面操作
vim /etc/sysconfig/network-scripts/ifcfg-eth0
修改完成后可以重启一下network的服务
service network restart

查找文件
格式:find 路径  -name 文件名
find ./ -name init.d 查看服务的配置信息,
这里只是演示一下查文件，要想查看服务的方法还要加上服务名,
例如
cat /etc/init.d/network
cat /etc/init.d/iptables
find ./ -name ifcfg-eth0

压缩文件
两种类型:tar.gz和zip
一般linux的压缩文件都是tar.gz
压缩文件
z:通过gzip过滤归档
c:创建一个新的归档
v:显示一个详细信息
f:指定压缩后的文件名/指定要解压缩的文件名
x:从归档中解出文件

tar -zcvf xxx.tar.gz one.txt
解压文件
tar -zxvf xxx.tar.gz

重要文件：
/etc/inittab linux的默认运行级别配置文件
/etc/sysconfig/network-script/ifcfg-eth0 网络配置文件
